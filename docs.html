A. Konsep Lua Inti (Global)

1. Table Hybrid & Tipe Data

**Pengertian:** Table adalah struktur data majemuk tunggal Lua yang berfungsi sebagai **Array** (indeks numerik) dan **Hash Map/Dictionary** (indeks string) secara bersamaan. Lua hanya memiliki delapan tipe data primitif, dan Table adalah kunci untuk semua struktur kompleks.

Contoh Code: Dualitas Table

local data = {
    "Peluru", -- Array key 1
    jumlah = 50, -- Map key "jumlah"
}
print(data[1])  -- Output: Peluru (Array Access)
print(data.jumlah) -- Output: 50 (Map Access)

2. Metatable & Metamethods

**Pengertian:** **Metatable** adalah *mechanism* untuk mendefinisikan perilaku table. **Metamethods** adalah fungsi yang disimpan di Metatable (seperti `__index` atau `__add`) yang dipanggil ketika operasi tertentu dilakukan pada table pemiliknya, memungkinkan OOP dan operator overloading.

Contoh Code: Metamethod `__add`

local vectorMt = { 
    __add = function(a, b) return {x = a.x + b.x} end
}
local v1 = setmetatable({x=1}, vectorMt)
local v2 = {x=2} 

print((v1 + v2).x) 
-- Output: 3 (Metamethod __add dipanggil)

3. Coroutines

**Pengertian:** Bentuk *multitasking* **kooperatif** (bukan paralel) yang memungkinkan fungsi untuk ditangguhkan (**`yield`**) dan dilanjutkan (**`resume`**) dari titik penghentiannya. Ideal untuk simulasi I/O, animasi, atau game loop berbasis *state machine*.

4. Closures & First-Class Functions

**Pengertian:** Di Lua, fungsi adalah objek **First-Class** (dapat disimpan di variabel, dilewatkan sebagai argumen). **Closure** adalah fungsi anonim yang menyimpan akses ke variabel lokal (`upvalue`) dari fungsi luarnya, bahkan setelah fungsi luar tersebut selesai.

5. Local vs Global Scope

**Pengertian:** Variabel **`local`** diakses langsung dan sangat cepat. Variabel **Global** (dideklarasikan tanpa `local`) diakses melalui *lookup* table `_G`, yang lebih lambat. Menggunakan `local` adalah kunci untuk kode Lua yang efisien.

6. Iterators (`ipairs`, `pairs`)

**Pengertian:** Fungsi generator yang digunakan dalam loop `for` generik. **`ipairs`** menjamin urutan numerik dan berhenti di `nil` pertama (Array). **`pairs`** melintasi semua kunci/nilai (Map) tanpa jaminan urutan.

7. Variadic Functions (`...`)

**Pengertian:** Fungsi yang dideklarasikan dengan elipsis (`...`) dan mampu menerima jumlah argumen yang tidak tetap. Argumen ini biasanya dikemas ke dalam table menggunakan sintaks `{...}`.

8. Tail Call Optimization (TCO)

**Pengertian:** Mekanisme penting di mana *stack frame* fungsi saat ini dihapus jika pemanggilan fungsi adalah **aksi terakhir**. Ini memungkinkan rekursi tak terbatas tanpa *stack overflow*, sering digunakan di *game engine*.

9. Garbage Collection (GC)

**Pengertian:** Sistem manajemen memori **inkremental** Lua. GC bekerja dalam tahapan kecil untuk mencegah *lag* besar yang disebabkan oleh pembersihan memori secara penuh (*stop-the-world*), vital untuk *frame rate* game yang stabil.

10. Error Handling (`pcall`)

**Pengertian:** Fungsi **`pcall`** (*protected call*) menjalankan fungsi yang diberikan dan menangkap kesalahan apa pun yang terjadi, mengembalikan status boolean dan pesan kesalahan. Ini mencegah skrip utama *crash*.

B. Konsep Lua Roblox Studio ðŸ‘¾

11. Instances & Datamodel

**Pengertian:** Semua objek di lingkungan Roblox (Player, Part, Model, Script) adalah **`Instance`**. **Datamodel** adalah struktur hirarki (*tree-like structure*) di mana semua Instance ini disimpan (`game.Workspace`, `game.Players`).

12. Events & Connection

**Pengertian:** Inti dari *event-driven programming* Roblox. Fungsi **`.Event:Connect(Function)`** akan menghubungkan fungsi ke *event* (misalnya, `Part.Touched`), menjalankan fungsi tersebut setiap kali *event* dipicu.

13. Server/Client Model

**Pengertian:** Pemisahan antara kode yang berjalan di **Server** (`Script`) yang mengontrol keadaan game universal, dan **Client** (`LocalScript`) yang mengontrol UI dan input pemain. Penting untuk keamanan dan sinkronisasi.

14. Remote Events/Functions

**Pengertian:** Mekanisme untuk komunikasi antar-proses (Server $\leftrightarrow$ Client). **RemoteEvents** digunakan untuk memicu tindakan satu arah, sedangkan **RemoteFunctions** menunggu respons balik.

15. Services & GetService

**Pengertian:** Modul bawaan Roblox yang menyediakan fungsionalitas utama. Mengaksesnya dengan **`game:GetService("ServiceName")`** adalah praktik terbaik untuk memastikan Service sudah dimuat.

16. Properties & Attributes

**Pengertian:** **Properties** adalah variabel bawaan pada Instance (`Part.Color`, `Player.Health`). **Attributes** adalah variabel kustom yang dapat ditambahkan developer ke Instance (misalnya, `Part:SetAttribute("Damage", 10)`).

17. Wait/Yielding

**Pengertian:** Fungsi **`task.wait(n)`** menangguhkan eksekusi skrip selama $n$ detik. Di Roblox, ini adalah bentuk *yielding* yang aman, karena skrip tidak memblokir *thread* utama.

18. Filtering Enabled

**Pengertian:** Konsep keamanan mendasar. Memastikan bahwa perubahan penting (seperti kerusakan atau posisi part) hanya dapat disahkan dan dimodifikasi oleh **Server**, mencegah *exploiting* dari sisi Client.

19. DataStoreService

**Pengertian:** Layanan yang digunakan untuk menyimpan data pemain secara persisten (permanen) di *server cloud* Roblox, kunci untuk menyimpan *progress*, skor, atau mata uang game.

20. RunService (Stepped/Heartbeat)

**Pengertian:** Service yang menyediakan *event* yang dipicu setiap *frame*. **`Heartbeat`** dipicu setelah simulasi fisika (ideal untuk *client-side*), sementara **`Stepped`** dipicu sebelum simulasi (ideal untuk *server-side*).
